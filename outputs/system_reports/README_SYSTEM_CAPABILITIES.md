# System Capabilities — Managed Governance Structure

This directory organizes TradeScan platform reports by core system capabilities rather than chronological run order. This facilitates rapid access to architectural, governance, and research documentation for developers and auditors.

## 01_system_architecture
Infrastructure and high-level platform design. Covers authority traces, pipeline flows, and multi-profile logic.

## 02_engine_core
Documentation regarding the core computation engine. Includes version-specific execution audits and validation results.

## 03_pipeline_orchestration
Master flow management. Covers decoupling strategies, implementation plans, and readiness status reports for research phases.

## 04_governance_and_guardrails
Enforcement documentation. Covers naming conventions, state-root separation, and infrastructure hardening summaries.

## 05_capital_and_risk_models
Financial engineering and risk management coverage. Includes sizing audits, safety wrappers, and dynamic pip value feasibility studies.

## 06_strategy_research
Documentation for discovery and cross-symbol verification. Covers filter schemas and collision audits.

## 07_system_complexity_reduction
Audit trail for the phased refactoring project. Tracks complexity reduction from baseline to final optimization.

## 08_baselines_and_archives
Authoritative snapshots of system state. Stores historical manifests and run records for regression testing.

## 09_experiment_logs
Specific trial run reviews and experimental separation plans. Includes engine reviews and go-live readiness assessments.
