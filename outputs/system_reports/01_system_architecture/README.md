# Architectural Documentation Guide

To understand the TradeScan system architecture, reading the documents in this directory is highly recommended. Please follow this specific order to build context from high-level boundaries down to low-level implementation details:

1. `SYSTEM_SURFACE_MAP.md` (System boundaries and architecture overview)
2. `pipeline_flow.md` (Execution flow and data orchestration)
3. `capability_map_analysis.md` (Specific stage capabilities and artifacts)
4. `SYSTEM_ENTRYPOINTS.md` (Operational entry points)
5. `PIPELINE_INVARIANTS.md` (Hard system invariants and determinism rules)
6. `REPOSITORY_AUTHORITY_MAP.md` (Deep-dive on directory and file authority)
