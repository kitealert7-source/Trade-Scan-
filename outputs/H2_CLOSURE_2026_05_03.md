## Linter extension

`tools/lint_encoding.py` previously only caught bare `.read_text()`. Extended to catch all three classes:

| Pattern | Behavior |
|---|---|
| `BARE_READ_TEXT` | `path.read_text()` with no args → flagged |
| `RE_WRITE_TEXT` | `path.write_text(...)` without `encoding=` in args → flagged |
| `RE_BUILTIN_OPEN` | `open(...)` in text mode without `encoding=` → flagged |

Special handling:
- **Binary modes** (`'rb'`, `'wb'`, `'ab'`, `'rb+'`, etc.) skipped — encoding doesn't apply.
- **`os.open()`** skipped via negative lookbehind — returns a file descriptor, not a stream.
- **Nested calls** in args (e.g. `open(Path("foo"), "r", encoding="utf-8")`) handled correctly.

The lint runs both as standalone scan (`python tools/lint_encoding.py`) and via `--staged` for pre-commit gating. The pre-commit hook already invokes it; no hook change needed.
